To 新生--研究生阶段的心得和建议

2019-09-10

研究生阶段是我在本科游历完成后继续进行修炼的一个阶段，2017年9月我怀着无比激动的心情进入了这个新学府，开始了我的新阶段，新起点。
会感到迷茫和无助，希望大家可以在任何时候都阅读这一篇我的文章，无论是正在研究还是说遇到困难的你们，都希望能给你们帮助。
研究生一年级都是在上课，只需要保证不挂科。当然，进组的修炼是必须的，就算老师日理万机，不在校内亲临指导，但是对自己弟子的关注不比那些老师在学校的学生少。我们组的机制和规范都必须遵循尚且还在完善中。我们要进行flip基础学习，有了基础知识之后才能跟着老师做研究，有了这样的研究铺垫，做起研究也会高效很多。
进入TULIP组，研究生该做的事情：
最重要的一句话：我们组内不需要叶公好龙的人，第一年的任务就是好好学习，等到研究方向会和你讨论你的研究方向
1.准备工作
老师说过，大脑是用来处理信息的，而不是存储信息的。因此，最最重要的工作就是学会给自己的研究写计划和学会如何与他人合作。
首先，必须要注册一个bitbucket账号，下载一个Smartgit，在我们组内公共区里会有bitbucket+smartgit 的配置方式，在FLIP学习需要自己push到bitbucket并分享到小组组长，记录自己的学习进度。
其次，PPR是必须要创建的，创建在石墨文档中，创建一个以自己名字的文件夹，两个文件PPR和Idea back，ppr包括了月计划和年计划，在钉钉的钉盘公共区中会有对应的介绍。
这两个准备工作，必须要提前准备。后面需要时可随时使用，通常在一个月的开始或最后讨论ppr。
2.FLIP学习
所有进入FLIP组的新人必须都会经历的事情就是FLIP学习，TULIP中有相对完善的学习材料，不管是概率论、矩阵分析还是机器学习深度学习等，都有最好的学习材料。珍惜这样的学习机会，不管是在研究还是以后的深造都有很大的帮助。一定要扎扎实实的学习，不要走回头路，培养自主学习能力。学习基础知识就像是一栋楼的地基需要稳固，才能建起高楼大厦，因此，前面学习扎扎实实后面研究才能胸有成竹。
3.阅读研究相关的书籍
彭老师写了一篇文章也出了一本书《研究生完全求生手册》，讲了如何做研究，完成一篇优秀文章需要的能力和分析技巧，以及文献回顾，通过批判性的眼光看他人的文章和从他人的文章中去寻找创新点，然后教你写文章。
同样地，我们组内公共区也有一本英文的书《Crafting Your Research Future》。
阅读了这些法宝，你就明白了如何做研究，也不会迷茫和徘徊。
4.如何搜索文章，和如果做paper presentation
进入一个领域最简单也最有效的方法就是找一本这个领域最早的论述专著或教材，这个可以向组内的师兄师姐进行请教，同时，在钉盘的公共区也有对应的查找文献指南，下载文章也有很好的帮助。
每次开会，所有要汇报的人员必须要准备PowerPoint，我们组内有对应的ppt模板，不要自己随便使用模板。
听别人的报告，听报告想方法可以解决自己的问题，要琢磨自己的方法能不能套上去。
例如，有五个问题，对应五个方法，就需要思考10次，联想到自己的东西，在那些文章中，要去思考：别人提的问题有没有意义，能不能提出类似的问题
5.心得建议
“学如逆水行舟，不进则退”，同样地，研究也一样，研究需要持之以恒，组内高质量的文章都是需要完整的研究过程，完整又系统的研究过程也可以帮助你培养好的学习习惯。
兴趣决定研究；可替代性；只有你找到感兴趣的点，你才会愿意挖掘下去。
基础知识大家都必须会。
为什么要有研究方法，
运动是为了该糟蹋身体的时候有的糟蹋。
减肥是为了看到想吃的东西的时候放开了吃。
做研究和带着研究帽子不一样，
无论你正在遭遇着什么，
你都要从落魄中站起来重振旗鼓，
要继续保持微笑，
就像从未受伤一样。
未来如何，在于我们能否“悟已往之不谏，知来者之可追”，并敏于践行。

一、concat：沿着一条轴，将多个对象堆叠到一起

concat方法相当于数据库中的全连接（union all），它不仅可以指定连接的方式（outer join或inner join）还可以指定按照某个轴进行连接。与数据库不同的是，它不会去重，但是可以使用drop_duplicates方法达到去重的效果。

concat(objs, axis=0, join='outer', join_axes=None, 
ignore_index=False,keys=None, levels=None, names=None, 
verify_integrity=False, copy=True):

pd.concat()只是单纯的把两个表拼接在一起，参数axis是关键，它用于指定是行还是列，axis默认是0。当axis=0时，pd.concat([obj1, obj2])的效果与obj1.append(obj2)是相同的；当axis=1时，pd.concat([obj1, obj2], axis=1)的效果与pd.merge(obj1, obj2, left_index=True, right_index=True, how='outer')是相同的。

concat参数介绍：
objs：需要连接的对象集合，一般是列表或字典；
axis：连接轴向；
join：参数为‘outer’或‘inner’；
join_axes=[]：指定自定义的索引；
keys=[]：创建层次化索引；
ignore_index=True：重建索引

举例：

df1=DataFrame(np.random.randn(3,4),columns=['a','b','c','d'])  
df2=DataFrame(np.random.randn(2,3),columns=['b','d','a'])    
pd.concat([df1,df2])

二、merge：通过键拼接列

类似于关系型数据库的连接方式，可以根据一个或多个键将不同的DatFrame连接起来。该函数的典型应用场景是，针对同一个主键存在两张不同字段的表，根据主键整合到一张表里面。

merge(left, right, how=’inner’, on=None, left_on=None, right_on=None,
left_index=False, right_index=False, sort=True,
suffixes=(‘_x’, ‘_y’), copy=True, indicator=False)
参数介绍：

left和right：两个不同的DataFrame；

how：连接方式，有inner、left、right、outer，默认为inner；

on：指的是用于连接的列索引名称，必须存在于左右两个DataFrame中，如果没有指定且其他参数也没有指定，则以两个DataFrame列名交集作为连接键；

left_on：左侧DataFrame中用于连接键的列名，这个参数左右列名不同但代表的含义相同时非常的有用；可以是列名或数组的长度等于长度综合

right_on：右侧DataFrame中用于连接键的列名；可以是列名或数组的长度等于长度综合

left_index：使用左侧DataFrame中的行索引作为连接键；在与多重 （层次） 的综合，级别数必须匹配联接键从右综合的数目

right_index：使用右侧DataFrame中的行索引作为连接键；在与多重 （层次） 的综合，级别数必须匹配联接键从右综合的数目

sort：默认为True，将合并的数据进行排序，设置为False可以提高性能；

suffixes：字符串值组成的元组，用于指定当左右DataFrame存在相同列名时在列名后面附加的后缀名称，默认为(‘_x’, ‘_y’)；

copy：默认为True，总是将数据复制到数据结构中，设置为False可以提高性能；即使重新索引是不必要总是从传递的综合对象，复制的数据 （默认值True）。在许多情况下不能避免，但可能会提高性能 / 内存使用情况。可以避免复制上述案件有些病理但尽管如此提供此选项。

indicator：显示合并数据中数据的来源情况,将列添加到输出综合呼吁 _merge 与信息源的每一行。_merge 是绝对类型，并对观测其合并键只出现在 ‘左’ 的综合，观测其合并键只会出现在 ‘正确’ 的综合，和两个如果观察合并关键发现在两个 right_only left_only 的值。

举例：

1.默认以重叠的列名当做连接键。

df1=DataFrame({'key':['a','b','b'],'data1':range(3)})    
df2=DataFrame({'key':['a','b','c'],'data2':range(3)})    
pd.merge(df1,df2)   #没有指定连接键，默认用重叠列名，没有指定连接方式
data1 key data2

github+hexo搭建博客遇到的问题

2018-11-28

1.hexo 发布之后 gitpage 自定义域名失效：

使用 gitpage 功能将博客托管在了 github 上，并配置 CNAME 将自己的域名解析了过去，在 github 的仓库设置中开始custom domain 的功能，这时候就可以直接使用自己的域名访问啦。

但是发现一个问题是，每次配置 custom domain 之后，再次hexo deploy 之后，custom domain 会被重置失效。

2.插入图片的方式：

1.首先把blog（hexo）目录下的_config.yml里的psot_asset_folder:设置为true

2.在blog（hexo）目录下执行:

npm install hexo-asset-image --save

3.在blog（hexo）目录下Git Bash Here，运行hexo n 博客名来生成md博客时，会在_post目录下看到一个与博客同名的文件夹。

4.将想要上传的图片先扔到文件夹下，然后在博客中使用markdown的格式引入图片：

!\[你想要输入的替代文字]\(xxxx/图片名.jpg)

ps：因为博客名和文件夹名字相同，所以不需要绝对路径，只要xxxx是文件夹的名字就可以了。

5.最后检查一下，hexo g生成页面后，进入public\2017\02\26\index.html文件中查看相关字段，可以发现，html标签内的语句是<img src="2017/02/26/xxxx/图片名.jpg">，而不是<img src="xxxx/图片名.jpg>。这很重要，关乎你的网页是否可以真正加载你想插入的图片。这就成功了，当然前面步骤操作正确的话，这一步也不用检查。

3.换电脑之后怎么解决：

原博客地址：换了电脑如何使用hexo继续写博客

操作步骤

一、安装必要软件

安装 Git 客户端

安装 node JS

二、在 github 官网添加新电脑产生的密钥

三、源文件拷贝

将你原来电脑上个人博客目录下必要文件拷到你的新电脑上（比如F:/Blog目录下），注意无需拷全部，只拷如下几个目录：

3. 山中无历日，寒尽不知年。

-研究的历程与时间表
硕士研究生的各位同学总是对研究的历程与该有的进度一无所知，因而紧张度日而难以入眠，或者轻松度过而浪费岁月。本章的目的，是简介研究的历程，与进度的合理安排。

研究主题的确定方法：针对你的候选的研究主题，逐一进行初步的文献回顾，以便于了解该主题的研究背景，包括问题的起源与背景，这个问题在现实世界与学术界的意义和价值（重要性），目前学术界比较有共识的主要结论，目前学术界悬而未解决的主要争议和癥结点，以及目前学术界的主要研究趋势和课题。希望这个探索的过程可以让你尽快下定决心，选定一个研究主题。

论文题目确定方法：在你选定的研究主题当中深化文献回顾，了解该主题内曾有过的各种研究子题、研究方法与研究设计，直到你对某一个具体问题有些感觉、有些想象。甚至有些模糊的主张或‘假说’，并且有兴趣进一步更深入地探讨它。这时候，你可以模仿读过的相关文献，以该领域内学术界惯用的术语和概念，给这个问题下一个标题，并且用一段话陈述你的问题意义与研究焦点，从而形成你的‘问题陈述’。

接着，针对你的论文题目，根据你过去的阅读经验，选择一组最恰当的关键词开始搜寻并筛选出与该问题密切相关的代表性论文，完成初步的文献搜寻，接着，借由文献之助，勾勒出学术界研究这个问题时所采取过的各种角度，研究的方法与流派，有共识与有争议的议题、各种主张和技术性细节，以及会影响到研究结论的所有可能因素，最后汇整处个流派的优缺点与得失，以及背后大致的原因和机制。这是一段交错着阅读、思索、分析、归纳与汇整的复杂过程。

研究生修炼手册（一）

2018-10-26

转眼间已经研究生二年级了，上了一年的研究生，感觉自己接触到了不少的东西，也学会了不少的东西，在学习上面，学习方式还是有待提高，在研究生阶段我们学会的是学习的能力和学习的方式，学习的东西或许在以后根本用不上，或许在未来的某一天某一个阶段，我们会慢慢的明白其中的深奥含义，而现在我们需要做的就是，持之以恒，学会学习。
学习是一个阶段也是一个过程，但是学无止境我们不能永远的停在学习的某一个特定的阶段 而停止寻找的脚步，当感到学习迷茫和碌碌无为的时候，就应该打开彭老师的这本书《研究生完全求生手册》。学会在以后的生活里，用别人的智慧解决自己的问题。

1. 达摩面壁，慧可立雪

-研究生学习会学到什么？

绝大多数研究生都知道，念研究生是为了要学习“做研究的方法”和“解决问题的方法”。但是，什么是“研究的方法”？毕业后有用吗？很多人不知该如何作答。此外，各个领域解决问题的方法有什么不同？

有些人说研究所学的东西以后不一定会有用；更有人干脆地说，研究所本来就是象牙塔，里面教的跟外面所需要学的根本不相干，毕业后当然不会有用。的确，硕博士班里面所学到的东西未来往往用不着，因为你毕业后的工作内容跟论文的研究问题可能毫无相干，但是，各个领域解决问题的方法和程序大同小异，所以你学到的研究方法可能一辈子受用。
不管是学术界或是其他领域，典型的研究工作有两种，其一是为一个重要的问题寻找严谨而可靠的答案，其次是为现存或未来的问题寻找更好的解决方案。而“研究方法”就是找出这些答案或解决方案的方法。
你可以用这些研究方法来解决职场上所遇到的问题，也可以用它来回答人生中许多重大的问题。

研究的目的是寻找正确的答案

如果你要为重大的问题寻找答案，你将需要一系列的方法和能力：（1）搜寻相关文献，研判文献的可靠性与品质，并迅速过滤品质较差的；（2）分析优质文献的观点、证据与推论过程，以确定它们的得失与短长；（3）比对不同文献的简介与相互的批评，以便找出所有文献中仍有待进一步理清的疑点；（4）汇整既有文献中可靠的部分，拼凑出事实的较完整面貌，并盘点其中尚待厘清的疑点；（5）对尚待厘清的部分进行第一手的调查或研究，以便获得问题的完整答案—-这就是研究的方法和完整的程序。
请注意，这五项能力的前四项都跟文献回顾或批判性思考有关，只有最后一项与创新有关。而且文献回顾并不只是要消极吸收既有的知识，更要积极对既有知识进行批判、汇整与评价，甚至还要从过去的文献寻找创新与突破的线索。所以，文献回顾在研究生工作中扮演着攸关成败的角色，它可以提升研究人员的批判能力，甚至还可以孕育出创新的策略，使创新变得比较容易，也可以降低研究过程的风险和成本。
当你在职场或人生中面对任何重大的问题、困扰或抉择，而必须在众说纷纭中找到正确的答案时，你就会发现论文研究过程中所学到的方法，可以帮助你找到最值得相信的答案，以及该采取保留态度的疑点。譬如，当你最关心的人不幸罹患癌症或不常见的疾病，而你又不确定地方医院的诊断与处方是否可信，也不知道是否需要劳师动众转移到病床严重不足的大医院，或者该找大医院的某一位医生时，你将会发现，研究生里学到的方法和能力可以帮你理清问题。

最省时省力的创新策略

研究的最终目的就是为了创新：学术界在寻找突破性、革命性的原创答案，企业界在寻找破坏性床照的新颖技术或方法，以便设计和生产出杀手级别的产品，垄断市场以获取最大的利润。
然而，创新的过程不可以完全仰赖灵感或运气，因为它们的不确定性和风险都太高，而且它们都是学不来的。
真正好的研究方法，是先从批判性的文献回顾，了解前人的失败经验和教训，以避免重蹈前人走过的陷阱和坎坷路，从而降低风险和缩短研究所需要的时间与资源；接着充分吸收前人的智慧，站在巨人的肩膀上，以便先立于不败之地；最后才是分析既有答案的缺点与不足之处，用自己的创意（聪明和运气）想出来改进的办法，以便突破既有，推陈出新。
“研究=站在巨人的肩膀上+创意”。先进行批判性的文献回顾，再用前人的经验与智慧降低研究的风险和成本，缩短完成计划的时间，并且提高研究的成果与品质。

总结

  • 研究的方法与目的就是为问题寻找正确或可靠的答案，或者为问题找到更好的解决方案。这一套方法与程序可以用来解决产业界、政府机构和学术界的问题，也可以用来解决人生中其他的关键问题。
  • 研究的过程始于文献回顾，创意是用来走完最后一哩路：先从文献回顾去了解前人的失败经验和教训，避免重蹈覆辙，借此降低风险；接着吸收前人的智慧，站在巨人的肩膀上，先立于不败之地；最后才是分析既有困难的缺点与不足处，用自己的创意（聪明和运气）相处改进的办法，以便推陈出新。
  • 先站在巨人的肩膀上，再把创新的资源集中在刀口上，这样才能充分发挥后发优势。

2. 黄金与粪土

-学术著作的要件与良窳

很多硕博士生都知道论文的品质攸关自己能否毕业，却不太清楚学位论文该具有的要件，以及如何衡量一本学术论文的品质。因为不知道这两个问题的答案，所以就搞不清楚自己为何而战，还经常误把粪土当黄金，将很多时间、心力浪费在“不具学术价值”的研究上。毕不了业或休业时间太长，往往都跟这个问题有密切关系。

学术著作的基本要件

（1）原创性与贡献：论文中包含的东西必须是学术界以前未曾了解的知识或解决方案，足以增进学术界对某个陌生领域的认识，或者深化学术界对既有问题的了解，或者对现有问题能提出新的有用的解决方案。对学术领域的进度有不容忽视的贡献或冲击。
（2）可靠的证据：论文中所提出来的方法或者解决方案必须有充分的证据支持，经过一层一层严谨的检验和对比，充分考虑到它的不准确性及对应的误差，充分考虑到分析方法的限制性和适用性，其可靠性经得起该领域专家的反复检测。
（3）批判性的检验：这些证据被可靠的理论架构整合在一起，严谨地分析和归纳它们所隐含各种可能的推论与矛盾，检验各种可能的假说与诠释后，剔除所有可被质疑的诠释或解释，然后保留经得起批判性检验的洞见、知识或解决方案，并严谨地论证合理性。
（4）理论性、系统性与一致性：这些洞见、知识或解决方案与前面的证据（也就是检验过程）形成一个前后一致、毫无矛盾的理论体系，有明确的前提、假设与适用条件，彼此之间有清楚的归纳与演绎关系，并且有具体的结论，可以被系统性地演绎出各种隐含的推论。因此，读者可以从不同角度进行查验其证据和论证过程，据以测试其有效性和优越性。
（5）客观性与可重复性：前面所说的理论体系陈述的事情，在明确的给定条件下，经得起学术界的反复检验，而排除了各种偶然性和个人的主观性。
（6）跟学术界的明确关系：上述的发现在现有知识体系中具有明确的位置与对话的对象，可以清楚的判断它们跟现有学术知识的继承关系、竞争关系和取代关系，可以用现有知识为基础去检验其有效性，也可以跟现有知识接轨而扩大其应用价值。
（7）文体的清晰性：整篇论文是以学术界惯用的概念、术语和文体写的，便于该领域专家阅读且积极避免各种可能的误会。其最终目标是可以不受读者所处时空与文化背景的影响，让具有专业能力的读者阅读后，活得完全一致的理解，而不会有任何理解上的歧义。

批判性思考与系统性论述

学术研究的三大能力：创新能力、自我批判能力与文献回顾的能力
学术论文必备的要点扼要摘述：（1）学术论文必须有原创性的发现与贡献，并且经过严谨地自我批判；（2）在明确给定的条件下，这些发现的有效性具有可重复性，经得起专家从各种角度的检验与批判，并借此排除偶然性和个人的主观性；（3）论文是以学术界惯用的概念、术语呵呵文体写的，清晰陈述各种发现、证据、论证与结论，便于读者的批判性阅读和各种角度的检视。
知其然却不知其所以然，在学术著作中，是不可以的，一定要搞清楚每个方面的前因后果，既能说服自己也能说服他人。将零碎而不相关的知识片段发展成系统性的知识，有好几个好处：首先，系统性的知识可以很快的被理解，可以提高知识的灵活性，其次，系统性知识中的隐含知识可以被很好的演绎出来，从而更容易的突出其内部的内在矛盾，也便于与不同来源的证据比对来确认其有效性。这就是自然科学比中世纪神学的宇宙观更可靠的原因。

系统性与客观性：从工业科学到人文科学

前面两节里对于客观知识与系统性知识的追求，在计量科学的相关研究领域都被普遍的接受。即使跨出理工院的大楼，在其他学院也是适用的，万变不离其宗，都希望研究成果能够具有‘不因人、时、地而异的有效性’，以及‘受过专业训练的人都不得不认同’的客观性。
然而绝大部分精神分析和后现代文化批判的著作还是一点也不像文学作品，他们都提供了丰富的证据以及极具说服力的论述，企图让读者接受他们的观点、论旨或结论，具有极强的批判性。
所有科学都要求研究生必须具备批判能力，随时保持批判性思考，尤其是要有能力自我批判，以摆脱主观的偏见、意识形态和独断论的影响，借此确保知识的客观有效性。

Net主题的配置

在 Hexo 中有两份主要的配置文件，其名称都是 _config.yml。 其中，一份位于站点根目录下，主要包含 Hexo 本身的配置；另一份位于主题目录下，这份配置由主题作者提供，主要用于配置主题相关的选项。

为了描述方便，在以下说明中，将前者称为 站点配置文件， 后者称为 主题配置文件。
